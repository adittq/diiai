# 奇门遁甲 × 64卦 信息拆解归档器

本技能用于把用户输入（问题、背景、对话片段、资料摘要）拆解为要素，按“奇门遁甲排盘”的结构组织分析，并把信息按“64卦”标准归档到本地 64 个文件夹中，最终给出结构化结论。

重要说明：奇门遁甲与64卦属于传统文化解释框架，本技能将其作为“信息组织与视角化推理”的工具，而非保证预测准确性。输出应提供可执行建议与不确定性提示。

## 何时调用

- 用户要求：按奇门遁甲/64卦方式分类、排盘、解盘、归档信息
- 需要：把零散信息体系化，并用固定模板生成分析结论
- 需要：把同类问题沉淀进可检索的 64 卦文件夹

## 输入

- `question`: 用户当前提问
- `context`: 背景信息（可为空）
- `timestamp`: 可选。若未提供，使用本地当前日期时间（仅用于一致性排序，不做神秘化解释）
- `domain`: 可选。场景标签（工作/情感/学习/健康/投资/法律/家庭等）

## 输出

1. `归档结果`：命中的卦与路径；写入的摘要文件名
2. `排盘要素`：九宫分布（九宫、九星、八门、八神、天盘干、地盘干）的文本化表示
3. `解盘结论`：用“取象→权衡→建议→风险→下一步”的结构回答用户
4. `现实建议`：不依赖玄学的可执行建议（至少 3 条）

## 本地目录规范（必须遵守）

在项目根目录创建并维护：

- `qimen64/`
  - `gua/`（64 卦文件夹）
    - `01_乾_Qian/`
    - `02_坤_Kun/`
    - …
    - `64_未济_WeiJi/`
  - `index.json`（可选：全局索引，记录条目元数据）

每次归档在命中的卦目录下新增一个条目文件：

- `YYYYMMDD_HHMMSS__<slug>.md`

文件内容包含：原始问题、要点拆解、排盘要素、结论摘要、标签。

如果目录不存在，先创建目录；若已存在，直接写入新条目文件。

## 分析流程（严格按顺序执行）

### 1) 信息拆解（Extraction）

从 `question + context` 中提取并输出：

- 目标：用户到底想得到什么（决策/建议/预测/解释/对比/计划）
- 约束：时间、资源、角色关系、禁忌、底线
- 变量：可控变量 / 不可控变量
- 风险：最坏情况、关键不确定因素
- 证据：用户明确给出的事实 vs 推测

### 2) 奇门要素建模（Qimen Schema）

在不依赖外部排盘库的情况下，使用“启发式排盘”（保证一致性与可复现）：

1. 生成 `seed`：对 `question + context + timestamp` 做稳定哈希（例如 sha256 的前若干位转整数）。
2. 用 `seed` 将以下要素分配到九宫（1-9）：
   - 九星（9 个）
   - 八门（8 个）
   - 八神（8 个）
   - 天盘干（10 取 9 或按规则裁剪）
   - 地盘干（10 取 9 或按规则裁剪）
3. 明确“用神/主线”：
   - 若是决策/取舍：以“门”为主线
   - 若是阻碍/卡点：以“宫位冲突/门克”为主线
   - 若是沟通/关系：以“神 + 门”为主线

输出一个文本化九宫表（每宫一行即可）：

`宫位: 星/门/神/天干/地干 | 含义关键词(1-3个)`

### 3) 64卦归类（Hexagram Classification）

将用户问题映射到 64 卦中的 1 卦（必要时给出 1 主卦 + 1 互卦/变卦作为补充，但归档只归到主卦目录）：

1. 先做“八卦维度打分”得到上卦/下卦（各取 1）：
   - 乾：主动/开创/规则
   - 坤：承载/配合/资源
   - 震：行动/起始/冲动
   - 巽：渗透/策略/影响
   - 坎：风险/不确定/陷阱
   - 离：清晰/传播/名誉
   - 艮：停止/边界/守成
   - 兑：交流/交易/喜悦

2. 用“关键词→维度”映射规则打分（可解释）：
   - 将提取到的 `目标/约束/变量/风险` 关键词映射到上述 8 维度（每个关键词可加 1-3 分）。
   - 得分最高的作为上卦，次高作为下卦（或按“风险维度为上卦、目标维度为下卦”的优先规则）。

3. 根据（上卦, 下卦）确定主卦编号。

输出：

- `主卦`: 编号 + 名称 + 简短取象（不超过 60 字）
- `判定依据`: 哪些关键词导致该卦（可复核）

### 4) 解盘与回答（Synthesis）

使用固定结构回答：

1. `局势判断`：用 2-4 句概括“现在是什么局”
2. `关键矛盾`：列出 2-3 条矛盾/约束
3. `三案并陈`：A 保守、B 折中、C 激进（每案含收益/代价/触发条件）
4. `风险与验证`：给出 3 条可验证的观察点（下周/下月能检查）
5. `一句话结论`：短句收束

### 5) 归档写入（Persistence）

1. 确保 `qimen64/gua/<主卦目录>/` 存在。
2. 生成条目文件名并写入 `.md`。
3. 可选：更新 `qimen64/index.json`，记录 `{id, ts, domain, gua_no, gua_name, tags, question_excerpt}`。

## 质量与安全约束

- 不把“某类人群”当作单一模板，避免刻板印象与冒犯性措辞
- 不输出违法、危险、或明显误导性建议
- 涉及医疗/法律/投资等高风险主题：必须提示专业咨询与信息核验
- 明确不确定性：哪些是事实、哪些是推断

## 示例调用（给模型看的）

输入：

- question: “我该不该换工作去创业？”
- context: “目前在大厂，想做餐饮连锁，但资金不多，家人反对。”

输出：

- 主卦目录 + 新建条目文件
- 九宫文本化排盘
- 三案并陈 + 风险验证点 + 一句话结论
